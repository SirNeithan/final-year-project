DATABASE UPDATES REQUIRED
=========================

Run these SQL commands to update your database:

1. Add phone column and make ZIP optional:
   
   mysql -u root -p smartdine < data/add_phone_column.sql

   OR manually run:
   
   USE smartdine;
   ALTER TABLE orders ADD COLUMN customer_phone VARCHAR(20) NULL AFTER customer_email;
   ALTER TABLE orders MODIFY COLUMN delivery_zip VARCHAR(20) NULL;

2. Add transaction_id column (if not already done):
   
   mysql -u root -p smartdine < data/add_transaction_id.sql

EMAIL CONFIGURATION
===================

IMPORTANT: Email sending may not work on localhost!

For emails to work properly:

OPTION 1: Local Testing (XAMPP/WAMP)
-------------------------------------
1. Configure sendmail in php.ini
2. Or use a tool like MailHog for testing

OPTION 2: Production Server
----------------------------
Emails should work automatically if your server has:
- Properly configured mail() function
- Valid domain name
- SPF/DKIM records set up

OPTION 3: Use SMTP (Recommended)
---------------------------------
For reliable email delivery, use PHPMailer with SMTP:

1. Install PHPMailer:
   composer require phpmailer/phpmailer

2. Update includes/email_functions.php to use PHPMailer

3. Configure SMTP settings (Gmail, SendGrid, etc.)

Example SMTP services:
- Gmail SMTP (free, limited)
- SendGrid (free tier available)
- Mailgun (free tier available)
- Amazon SES (pay as you go)

EMAIL TESTING
=============

To test if emails are being sent:

1. Check PHP error log:
   - XAMPP: xampp/php/logs/php_error_log
   - Linux: /var/log/php_error.log

2. Look for lines like:
   "Order confirmation email sent successfully for order #123"
   OR
   "Failed to send order confirmation email for order #123"

3. Check your email inbox (and spam folder!)

TROUBLESHOOTING
===============

Q: Emails not being sent?
A: Check PHP error log. On localhost, mail() often doesn't work.
   Solution: Use SMTP with PHPMailer or test on production server.

Q: Order created but no email?
A: The order will still be created even if email fails.
   This is intentional - we don't want to lose orders due to email issues.

Q: Getting "ZIP code required" error?
A: Clear your browser cache and refresh the page.
   Make sure you ran the database migration.

Q: Phone number not saving?
A: Run the database migration: data/add_phone_column.sql
